## 1. 项目背景与目标

### 1.1 问题陈述
体育赛事参与者在下注决策时普遍存在三个痛点：
1. **凭感觉决策**：缺乏量化依据，无法判断某个赔率是"贵"还是"便宜"。
2. **不懂资金管理**：不知道每个标的该投多少，容易重仓单一结果或均摊浪费。
3. **看不清庄家抽水**：不理解固定奖金赔率里内置的返还率损失，长期亏损而不自知。

### 1.2 产品目标
构建一个**决策辅助工具**（非稳赢工具），用统计模型和市场有效性分析，帮助用户：
- 量化每个赛果/比分的发生概率；
- 识别赔率被高估的"正期望标的"（value bet）；
- 用科学的资金管理（凯利公式）分配投注额。

### 1.3 非目标（Non-Goals）
- ❌ 不承诺盈利，不提供"稳赢/必中"预测。
- ❌ 不做实时数据抓取（MVP 阶段数据手动输入）。
- ❌ 不涉及任何真实资金交易或支付。

### 1.4 成功指标（North Star & 输入指标）
| 层级 | 指标 | 目标 |
|---|---|---|
| North Star | 周活跃使用次数 | 验证需求真实性 |
| 输入指标 | UV、核心功能使用率、页面停留时长 | 埋点追踪 |
| 健康指标 | 跳出率、"一键同步"转化率 | 迭代优化 |

## 2. 用户与场景

### 2.1 目标用户
- **主要用户**：有一定数据素养、希望理性决策的体育赛事参与者。
- **次要用户**：对概率/量化建模感兴趣的学习者（工具具备教学价值）。

### 2.2 核心使用场景
> 用户小 A 想在某场比赛下注，但不确定买什么、买多少。
> 1. 打开精算分析器，填入两队近期进失球 → 自动算出预期进球 λ；
> 2. 填入赔率 → 工具算出各盘口模型概率、市场概率、EV，高亮 value bet；
> 3. 勾选想买的标的 → 一键同步到分注计算器；
> 4. 填入总预算 → 凯利公式给出每注建议金额。

## 3. 功能需求

### 3.1 功能地图
```
系统
├── 工具一：精算分析器 (predictor.html)
│   ├── ① 预期进球 λ 输入
│   ├── 🧮 λ 快速估算器（近期数据 → λ）
│   ├── ② 胜平负盘：去抽水 + 模型对比 + EV + 融合概率
│   ├── ③ 总进球盘：同上
│   ├── ④ 精确比分盘：同上（8×8 矩阵）
│   ├── 🎯 Value Bet 汇总
│   └── 一键同步（含重叠检测防呆）
│
└── 工具二：分注计算器 (index.html)
    ├── 模式：按总预算 / 按目标收回
    ├── 分配策略：均衡收回 / 按看好程度 / 凯利 / 手动输入
    ├── 2 的倍数凑整 + 差额提示
    ├── 期望值计算
    └── 接收同步数据 → 自动凯利生成购买建议
```

### 3.2 功能详述

#### F1 · λ 快速估算器
- **输入**：主/客队近 N 场的总进球、总失球、场次；主场系数；收缩比例。
- **输出**：攻防强度、基准场均、估算 λ₁/λ₂，一键应用到模型。
- **验收**：西班牙(9/0,5) vs 比利时(12/5,5) 中立场 30%收缩 → λ₁≈1.38、λ₂≈0.62。

#### F2 · 三盘口概率与 EV 分析
- **输入**：各盘口赔率。
- **输出**：每标的的模型概率、市场隐含概率（去抽水）、EV/元、value 标记；各盘抽水率。
- **验收**：去抽水后概率之和 = 100%；EV = 模型概率 × 赔率 − 1。

#### F3 · 一键同步 + 重叠检测
- **输入**：勾选的标的。
- **规则**：检测「比分 ⊂ 总进球」「比分 ⊂ 胜平负」「总进球0 ⊂ 平」三类重叠，弹窗提示。
- **输出**：通过 localStorage 传递到分注计算器，自动选凯利策略。

#### F4 · 分注计算器四种分配策略
| 策略 | 公式 | 适用 |
|---|---|---|
| 均衡收回 | 投注额 ∝ 1/赔率 | 无判断，求命中收回一致 |
| 按看好程度 | 投注额 ∝ 我估概率 | 有倾向 |
| 凯利公式 | f = (b·p−q)/b | 有真实优势，求长期最优 |
| 手动输入 | 用户自填 | 完全自主 |

## 4. 【核心】数学模型详述

> 本节是系统的算法核心，写到可复现深度。

### 4.1 模型总览
系统采用**分层建模**思路。核心哲学（v2.0 修正）：**国际锐利盘（Pinnacle 等）已聚合全部信息，是最优的"真实概率"来源；我们不与之竞争，而是以它为真值反推 λ，再评估软盘（竞彩）的命中机会。**
```
数据层：多博彩赔率(The Odds API) + 伤停/交锋(API-Football) + xG(understat)
第1层：λ 估计   → 【主】赔率反推求解器  /【辅】强度分解 /【验证】xG 交叉比对
第2层：比分分布 → Poisson × Dixon-Coles 修正 → 8×8 矩阵
第3层：盘口聚合 → 胜平负/总进球/比分/让球概率
第4层：决策输出 → 命中概率排序(胜负/总进球/比分/Top-N覆盖) + 去抽水EV参考
```

#### 4.1.1 λ 的两种来源（v2.0 新增反推法）
- **【主】赔率反推**（4.2A）：从锐利盘的胜平负+大小球+亚盘反解 λ。市场信息含量远超任何手工数据。
- **【辅】强度分解**（4.2B）：无赔率时，用两队近期进失球估 λ。
- **【验证】xG 交叉比对**：用两队近期场均 xG 校验反推的 λ，一致则置信度高，分歧提示市场定价了 xG 外信息（伤停/战意）。

---

### 4.2A 第 1 层【主】：从赔率反推 λ（v2.0 核心方法）

**动机**：博彩公司的锐利盘赔率已聚合全球资金与信息，其隐含概率是目前可得的最佳"真实概率"。与其用滞后、片面的历史数据估 λ，不如**反解出能最好复现市场赔率的 λ**。

**求解流程**：
1. 对每个盘口去抽水（4.5.1），得市场隐含概率：胜平负 (P_W, P_D, P_L)；每条大小球盘线的 P(总进球>line)；每条让球盘线的 P(主队让 point 后赢盘)。
2. 定义模型在给定 (λ₁, λ₂, ρ) 下对上述所有量的预测。
3. **网格搜索**最小化加权平方误差：
$$(\lambda_1^*, \lambda_2^*) = \arg\min \Big[ (P_W^{mdl}-P_W)^2 + (P_D^{mdl}-P_D)^2 + (P_L^{mdl}-P_L)^2 + \frac{1}{n_T}\sum_{k}(P_{O,k}^{mdl}-P_{O,k})^2 + \frac{1}{n_S}\sum_{m}(P_{C,m}^{mdl}-P_{C,m})^2 \Big]$$
   其中大小球/让球各盘线误差按盘线数均摊，避免盖过胜平负主约束。
4. **两阶段搜索**：先粗搜（λ∈[0.2,3.5]，步长 0.1），再在最优点邻域细搜（步长 0.02）。
5. 约束数 = 3（胜平负）+ n_T（大小球盘线）+ n_S（让球盘线），实测常达 9+ 个，λ 估计高度稳健（拟合误差 < 0.01）。

> 关键工程点：The Odds API 单次调用即返回某盘口的**全部盘线**，故用全部盘线做约束**不额外消耗 API 额度**。

### 4.2B 第 1 层【辅】：强度分解估 λ（无赔率时）

#### 4.2.1 理论基础
足球比赛中，一支球队的进球能力可分解为**自身进攻强度**与**对手防守弱度**的乘积效应（Maher 1982；Dixon-Coles 1997）。

#### 4.2.2 强度计算
设：
- `μ` = 赛事基准场均进球（单队）
- 主队场均进球 `h_gf`、场均失球 `h_ga`
- 客队场均进球 `a_gf`、场均失球 `a_ga`

**基准**（MVP 用两队数据代理赛事均值）：
$$\mu = \frac{h_{gf} + h_{ga} + a_{gf} + a_{ga}}{4}$$

**攻防强度**（相对基准的倍数）：
$$\text{主队进攻力 } Att_h = \frac{h_{gf}}{\mu}, \quad \text{主队防守弱度 } Def_h = \frac{h_{ga}}{\mu}$$
$$\text{客队进攻力 } Att_a = \frac{a_{gf}}{\mu}, \quad \text{客队防守弱度 } Def_a = \frac{a_{ga}}{\mu}$$
> 强度 = 1 表示平均水平；>1 进攻强/防守漏；<1 进攻弱/防守稳。

#### 4.2.3 贝叶斯收缩（防过拟合）
小样本（如淘汰赛仅 2-5 场）下强度易被极端值带偏（如"零封"使对手进攻力被压到 0）。用收缩系数 `s`（默认 0.30）把强度向平均值 1.0 拉回：
$$\text{shrink}(x) = x \cdot (1 - s) + 1.0 \cdot s$$

#### 4.2.4 预期进球
$$\lambda_1 = \mu \cdot \text{shrink}(Att_h) \cdot \text{shrink}(Def_a) \cdot \gamma_{home}$$
$$\lambda_2 = \mu \cdot \text{shrink}(Att_a) \cdot \text{shrink}(Def_h) \cdot \gamma_{away}$$
其中主场系数 `γ_home`：中立场 = 1.0，主场 = 1.1；客队镜像 `γ_away = 2 − γ_home`。

#### 4.2.5 时间衰减（完整版；MVP 简化为等权）
理论上越近的比赛越能反映当前状态，对第 `i` 场（距今 `n−1−i`）赋权：
$$w_i = \xi^{\,n-1-i}, \quad \xi = 0.85$$
加权场均 = Σ(该场数据 × wᵢ) / Σwᵢ。MVP 阶段用等权汇总（近 N 场求和），影响较小。

### 4.3 第 2 层：比分概率分布（Poisson + Dixon-Coles）

#### 4.3.1 泊松基础
假设各队进球数服从泊松分布：
$$P(X = k) = \frac{\lambda^k e^{-\lambda}}{k!}$$
主队进 `i` 球、客队进 `j` 球（先假设独立）：
$$P(i, j) = P_{\lambda_1}(i) \cdot P_{\lambda_2}(j)$$

#### 4.3.2 Dixon-Coles 低比分修正（关键改进）
裸泊松独立假设会**系统性低估低比分**（0:0、1:0、0:1、1:1），因为现实中比分低时两队进球呈负相关（防守对抗、比赛节奏）。引入修正因子 τ：
$$\tau(i,j) = \begin{cases} 1 - \lambda_1 \lambda_2 \rho & (i,j)=(0,0) \\ 1 + \lambda_1 \rho & (i,j)=(0,1) \\ 1 + \lambda_2 \rho & (i,j)=(1,0) \\ 1 - \rho & (i,j)=(1,1) \\ 1 & \text{其他} \end{cases}$$
其中 `ρ` 为相关系数，经验取 `-0.03 ~ -0.12`（默认 -0.08）。

修正后比分概率：
$$P(i,j) = \tau(i,j) \cdot P_{\lambda_1}(i) \cdot P_{\lambda_2}(j)$$

#### 4.3.3 归一化
对 8×8 矩阵（i,j ∈ [0,7]）归一化，使 ΣP(i,j) = 1：
$$P(i,j) \leftarrow \frac{P(i,j)}{\sum_{i,j} P(i,j)}$$

---

### 4.4 第 3 层：盘口概率聚合

从比分矩阵汇总：

**胜平负**：
$$P(\text{主胜}) = \sum_{i>j} P(i,j), \quad P(\text{平}) = \sum_{i=j} P(i,j), \quad P(\text{主负}) = \sum_{i<j} P(i,j)$$

**总进球数**（进 g 球）：
$$P(\text{总进球}=g) = \sum_{i+j=g} P(i,j)$$

**精确比分**：直接取 P(i,j)。

**淘汰赛晋级**（含加时/点球，平局按实力分配）：
$$P(\text{主队晋级}) = P(\text{主胜}) + P(\text{平}) \cdot \theta$$
其中 θ 为强队在平局局面下的晋级倾向（默认约 0.58）。

### 4.5 第 4 层：决策输出

#### 4.5.0 命中概率导向（v2.0 核心）
固定奖金玩法本质是"**中了按赔率得分，不中即 0**"——不存在反复套利，用户对手是**比赛的不确定性**而非庄家收益率。因此输出**以命中概率排序**，按颗粒度给用户选择：
- **押胜负**：命中率最高（约 50–65%），赔率最低。
- **押总进球**：Top3 各带概率，中等颗粒度。
- **押精确比分**：Top5 各带概率，命中率低（单个约 8–13%）、赔率高。
- **覆盖策略**：同买 Top3 / Top5 比分，累计命中率（如 Top3≈37%），配合分注计算器分配金额。

EV/value 作为**次要参考**保留（下），但排序主看命中率。

#### 4.5.1 去抽水（de-vigging）
博彩赔率的倒数之和 > 1，超出部分为抽水（overround / margin）：
$$\text{margin} = \sum_k \frac{1}{O_k} - 1$$
市场隐含真实概率（归一化）：
$$P_{market}(k) = \frac{1/O_k}{\sum_m 1/O_m}$$

#### 4.5.2 期望值（EV）与 Value Bet
$$EV = P_{model} \cdot O - 1$$
- EV > 0：正期望，赔率被高估，value bet。
- 系统标记 EV > 0.05 为 ★VALUE。

#### 4.5.3 模型-市场融合
单一模型难持续战胜聚合全市场信息的赔率，故加权融合（市场权重 W=0.70）：
$$P_{fuse}(k) = W \cdot P_{market}(k) + (1 - W) \cdot P_{model}(k)$$
再归一化。

#### 4.5.4 凯利公式（仓位管理）
最优下注比例：
$$f^* = \frac{b \cdot p - q}{b}$$
其中 `b = 赔率 − 1`，`p` = 估计胜率，`q = 1 − p`。
- f* ≤ 0：无优势，不下注。
- 实战建议用**半凯利**（f*/2）降低波动。

### 4.6 模型局限与已知偏差（诚实披露）
1. **基准代理偏差**：MVP 用两队数据代理赛事均值，强强/弱弱对话时基准会偏高/偏低。
2. **进球独立性假设**：Dixon-Coles 只修正了低比分，高比分仍假设独立。
3. **λ 依赖输入质量**："垃圾进垃圾出"，近期数据的代表性直接决定预测质量。
4. **不含情境因素**：伤停、红牌、天气、赛程密度等未纳入，需用户在 λ 上手动调整。
5. **静态模型**：不做赛中动态更新。

## 5. 技术实现

### 5.1 架构
```
前端(GitHub Pages)  ──▶  Cloudflare Worker 代理  ──▶  The Odds API / API-Football
  概率建模+决策            隐藏密钥·解决CORS·聚合            锐利盘赔率 / 伤停·交锋
       ◀──────────────────── JSON ────────────────────◀
  xG(understat 手录) ──▶ 交叉验证 λ
```
- **纯前端单页应用**：所有建模计算在浏览器完成，零建模后端。
- **Serverless API 代理**（Cloudflare Workers）：解决三问题——① 隐藏 API 密钥（不暴露前端）② 解决 CORS 跨域 ③ 聚合多博彩赔率取均值。
- **多源数据集成**：The Odds API（赔率）+ API-Football（见下表）+ xG（验证），密钥经 `wrangler secret` 注入，绝不出现在前端。
- **数据传递**：localStorage 实现跨页面（精算器↔分注计算器）同步。
- **部署**：GitHub → GitHub Pages 自动 CI/CD；Worker 经 wrangler 部署。用户自助——页内提供代理申请与部署图文说明，任何人可 fork 后 10 分钟接入自己的 key。
